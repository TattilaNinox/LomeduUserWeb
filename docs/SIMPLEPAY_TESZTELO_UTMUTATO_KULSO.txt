3.1 TESZT: Sikeres Fizetés (9.6.1)
══════════════════════════════════════════════════════════════════

LÉPÉSEK:
  1. "Előfizetés megújítása" gomb megnyomása
  
  2. Megjelenik: "Adattovábbítási nyilatkozat" ablak
     - Olvassa el a nyilatkozatot
     - Jelölje be az "Elfogadom" checkbox-ot
     - Kattintson az "Elfogadom és fizetek" gombra
  
  3. Átirányítás a SimplePay fizetőoldalra
  
  4. Teszt bankkártya adatok megadása:
     ┌─────────────────────────────────────┐
     │ Kártyaszám: 4000 0000 0000 0002     │
     │ Lejárat: 12/26 (vagy bármely jövő)   │
     │ CVV: 123                             │
     │ Név: Test User                       │
     └─────────────────────────────────────┘
  
  5. "Fizetés" gomb megnyomása
  
  6. Várakozás (1-2 másodperc)
  
  7. Automatikus visszairányítás a Lomedu oldalra

ELVÁRT EREDMÉNY:
  ✅ Megjelenik egy párbeszédablak:
     - Cím: "Sikeres tranzakció" (zöld pipa ikon)
     - Szöveg: "A fizetés sikeresen megtörtént!"
     - Megjelenik: SimplePay tranzakcióazonosító
  
  ✅ A Fiók adatok oldalon:
     - Előfizetési státusz: "Prémium előfizetésed aktív"
     - Lejárati dátum: 30 nappal későbbi dátum
  
  ✅ A "Fizetési előzmények" táblázatban:
     - Új sor jelenik meg automatikusan
     - Státusz: "Sikeres" (zöld chip)
     - Összeg: 4,350 Ft
     - Leírás: "30 napos előfizetés"


3.2 TESZT: Sikertelen Fizetés (9.6.2)
══════════════════════════════════════════════════════════════════

ELŐKÉSZÜLET:
  Először állítsa vissza az állapotot:
  - Görgessen le a lap aljára
  - "Teszt eszközök" szekció
  - Kattintson a "Reset (ingyenes állapot)" gombra (kék)
  - Erősítse meg a "Visszaállítás" gombbal

LÉPÉSEK:
  1-3. Ugyanaz mint a sikeres fizetésnél (adattovábbítási nyilatkozat elfogadás)
  
  4. Sikertelen teszt bankkártya megadása:
     ┌─────────────────────────────────────┐
     │ Kártyaszám: 4000 0000 0000 0119     │
     │ Lejárat: 12/26                       │
     │ CVV: 123                             │
     └─────────────────────────────────────┘
  
  5. "Fizetés" gomb
  
  6. A bank elutasítja a tranzakciót
  
  7. Automatikus visszairányítás

ELVÁRT EREDMÉNY:
  ✅ Megjelenik részletes párbeszédablak:
     - Cím: "Sikertelen tranzakció" (piros hiba ikon)
     - SimplePay tranzakcióazonosító (piros háttérrel kiemelve)
     - Szöveg: "Kérjük, ellenőrizze a tranzakció során megadott 
                adatok helyességét."
     - Banki kapcsolatfelvételi javaslat
     - Lehetséges okok felsorolása (általános)
     - "Újrapróbálás" gomb
  
  ✅ A "Fizetési előzmények" táblázatban:
     - Új sor jelenik meg AUTOMATIKUSAN (1-2 mp)
     - Státusz: "Sikertelen" (piros chip)
  
  FONTOS ELLENŐRZENDŐ (SimplePay 3.13.3 Spec):
     ✅ SimplePay tranzakcióazonosító LÁTHATÓ
     ✅ Részletes tájékoztatás van
     ✅ Banki kapcsolatfelvételi javaslat van
     ✅ NINCS konkrét ok (pl. "limittúllépés") - csak általános


3.3 TESZT: Időtúllépés (9.6.3)
══════════════════════════════════════════════════════════════════

ELŐKÉSZÜLET:
  "Reset (ingyenes állapot)" gomb (Teszt eszközök)

LÉPÉSEK:
  1-3. Fizetés indítás, SimplePay oldalra átirányítás
  
  4. A SimplePay fizetőoldalon: NE adjon meg semmit!
  
  5. Várjon 30 percet (ez a timeout beállítás)
  
  6. Automatikus visszairányítás 30 perc után

ELVÁRT EREDMÉNY:
  ✅ Megjelenik párbeszédablak:
     - Cím: "Időtúllépés" (narancs óra ikon)
     - Szöveg: "Ön túllépte a tranzakció elindításának lehetséges 
                maximális idejét."
     - Magyarázat: "A fizetési időkeret (30 perc) lejárt..."
     - Biztosítás: "Nem történt pénzügyi terhelés" (zöld dobozban)
  
  ✅ A "Fizetési előzmények" táblázatban:
     - Státusz: "Időtúllépés" (narancs chip)
  
  FONTOS (SimplePay 3.13.2 Spec):
     ✅ NINCS SimplePay tranzakcióazonosító (nem történt fizetési kísérlet)
     ✅ NINCS "sikertelen fizetés" szöveg
     ✅ Biztosítás van a nem történt terhelésről

5. FIZETÉSI ELŐZMÉNYEK - AUTOMATIKUS FRISSÍTÉS
================================================================================

A "Fizetési előzmények" táblázat a SimplePay logo alatt található.

MŰKÖDÉS:
  - Automatikusan frissül (nincs szükség F5-re)
  - Minden új fizetés azonnal megjelenik
  - Státusz változások valós időben látszanak

TÁBLÁZAT OSZLOPOK:
  - Dátum: Fizetés időpontja
  - Leírás: "30 napos előfizetés"
  - Összeg: 4,350 Ft
  - Státusz: Színes chip (Sikeres/Sikertelen/stb.)

STÁTUSZ TÍPUSOK:
  🟢 Sikeres - Sikeres fizetés
  🔴 Sikertelen - Elutasított fizetés
  🟡 Folyamatban - Még feldolgozás alatt
  🟠 Időtúllépés - 30 perc eltelt
  ⚪ Megszakított - Vissza gomb/bezárt böngésző

6. TESZTELÉSI FORGATÓKÖNYV (Ajánlott Sorrend)
================================================================================

TESZT 1: Sikeres Fizetés
  1. Reset gomb
  2. Fizetés indítás
  3. Teszt kártya: 4000 0000 0000 0002
  4. Ellenőrizze a sikeres dialógot
  5. Ellenőrizze a fizetési előzményeket (Sikeres chip)

TESZT 2: Sikertelen Fizetés
  1. Reset gomb
  2. Fizetés indítás
  3. Teszt kártya: 4000 0000 0000 0119
  4. Ellenőrizze a sikertelen dialógot (SimplePay ID, banki javaslat)
  5. Ellenőrizze a fizetési előzményeket (Sikertelen chip)

TESZT 3: Megszakított Fizetés
  1. Reset gomb
  2. Fizetés indítás
  3. SimplePay oldalon: "Vissza" gomb
  4. Ellenőrizze a megszakított dialógot
  5. Ellenőrizze: NINCS SimplePay ID, van biztosítás

TESZT 4: Időtúllépés
  1. Reset gomb
  2. Fizetés indítás
  3. NE adjon meg kártyaadatokat 30 percig
  4. Ellenőrizze az időtúllépés dialógot
  5. Ellenőrizze: NINCS SimplePay ID, van biztosítás

TESZT 5: SimplePay Logo
  - Görgessen le a Fiók adatok oldalon
  - Ellenőrizze: SimplePay logo látható

TESZT 6: Adattovábbítási Nyilatkozat
  - Fizetés indításnál megjelenik
  - Checkbox elfogadás szükséges
  - Csak elfogadás után lehet fizetni
